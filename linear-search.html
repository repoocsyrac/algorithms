Linear Search

Key facts

  • Goal: Finds specific element in a list.
  • Best-case time complexity: O(1)
  • Worst-case time complexity: O(N)
  • Average time complexity: O(N)
  • Space complexity: O(1)
  • Pros: Can be used when list is unsorted, can be performed in-place
  • Cons: Poor time complexity
  • When to use: When you are searching a very small list

How it works

Starting at the first element of the list, we go through each element in turn, checking whether or not it is the element we're looking for.

If so, the index the element was found at is returned.

If we reach the end of the list and no index has been found, we have determined that the element is not in the list.
